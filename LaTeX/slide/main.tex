\tightframe{O problema}
\begin{columns}[T,totalwidth=\textwidth]
\column{.46\textwidth}
{\Large\bfseries A base não entrega caixas.}

\vspace{.35cm}
\begin{itemize}
  \item A anotação disponível é o ponto nuclear.
  \item O ponto indica centro, mas não extensão do objeto.
  \item Detectores precisam de uma região de referência para aprender localização.
\end{itemize}

\vspace{.35cm}
\smallcard{ufpiBlue}{5.0cm}{Questão central}{Como converter pontos em um protocolo de detecção e contagem que possa ser auditado cientificamente?}

\column{.48\textwidth}
{\Large\bfseries Por que isso importa?}

\vspace{.08cm}
\begin{itemize}
  \item A caixa sintética define o alvo espacial do detector.
  \item Caixas pequenas podem perder vizinhança visual.
  \item Caixas grandes podem criar sobreposição e supercontagem.
  \item O tamanho precisa ser escolhido e reportado.
\end{itemize}

\vspace{.08cm}
\smallcard{ufpiGold}{5.6cm}{Decisão adotada}{O sweep selecionou 144 pixels como tamanho operacional para a validação cruzada final.}

% 16
\tightframe{Conclusões}
\begin{columns}[T,totalwidth=\textwidth]
\column{.58\textwidth}
{\Large\bfseries Conclusão científica}
\vspace{.15cm}
\begin{itemize}
  \item Pontos nucleares podem sustentar detecção e contagem quando o protocolo explicita a geometria usada.
  \item O sweep justificou 144 pixels como escolha operacional, não como tamanho universal.
  \item A validação cruzada 5-fold estima o desempenho a partir dos cinco testes externos.
  \item O detector mantém bom recall, mas exige atenção a falsos positivos e supercontagem.
\end{itemize}
